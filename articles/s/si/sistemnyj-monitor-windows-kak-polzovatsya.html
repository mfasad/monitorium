Системный монитор Windows: полное руководство по использованию в 2026 году

Системный монитор Windows (или Performance Monitor) — это мощный встроенный инструмент, который позволяет отслеживать производительность компьютера в реальном времени, диагностировать проблемы и оптимизировать работу системы. Многие пользователи даже не подозревают о его существовании, предпочитая сторонние утилиты вроде HWMonitor или AIDA64. Между тем, системный монитор способен предоставить не менее детальную информацию — от загрузки процессора до активности жестких дисков — и при этом не требует установки дополнительного ПО.

В этой статье мы разберём, как запустить и настроить системный монитор, какие параметры стоит отслеживать для диагностики типичных проблем (например, подвисаний или перегрева), и как создать собственные отчёты для анализа производительности. Особое внимание уделим практическим сценариям: от проверки игры на лаги до поиска "прожорливых" процессов, которые тормозят систему. Если вы когда-либо сталкивались с замедлениями Windows 10/11 или хотите просто лучше понять, как работает ваш ПК — этот гайд для вас.

Как открыть системный монитор в Windows: 4 простых способа

Запустить Performance Monitor можно несколькими способами — выберите наиболее удобный для вас. Все методы работают в Windows 10 и Windows 11, за исключением редких сборок с урезанным функционалом (например, Windows 10 LTSC).

  • 🔍 Через поиск: нажмите Win + S, введите Системный монитор (или Performance Monitor на английской версии) и выберите найденное приложение.
  • 📁 Через "Выполнить": нажмите Win + R, введите perfmon и нажмите Enter.
  • 🖥️ Через "Панель управления": перейдите в Панель управления → Администрирование → Системный монитор.
  • 📊 Через "Диспетчер задач": откройте диспетчер (Ctrl + Shift + Esc), перейдите на вкладку Производительность и нажмите Открыть монитор ресурсов, а затем — Системный монитор.

После запуска вы увидите главное окно с тремя основными разделами: Системный монитор, Журналы и оповещения производительности и Отчёты. Начинающим пользователям достаточно первых двух вкладок — остальные функции пригодятся для глубокого анализа.

Основные элементы интерфейса: что означают графики и показатели

Интерфейс системного монитора может показаться перегруженным, но на самом деле он логично структурирован. Главное окно разделено на несколько ключевых зон:

  • 📈 График в реальном времени: отображает выбранные показатели (по умолчанию — % загрузки процессора). Ось X — время, ось Y — значение параметра.
  • 📋 Панель инструментов: кнопки для добавления счётчиков, создания отчётов и настройки отображения.
  • 📊 Легенда: список активных счётчиков с цветовой маркировкой (чтобы отличать линии на графике).
  • ⚙️ Свойства: здесь настраиваются интервал обновления данных, стиль графика и единицы измерения.

По умолчанию системный монитор показывает только % загрузки процессора, но это лишь вершина айсберга. Чтобы добавить другие параметры, нажмите на кнопку + (зелёный плюс) на панели инструментов. Откроется окно со списком доступных счётчиков, сгруппированных по категориям: Процессор, Память, Диск, Сеть и т.д. Например, для диагностики тормозов игры полезно добавить счётчики GPU Engine (из категории Адаптер видеокарты) и % используемой памяти (из Память).

Какие счётчики добавить для диагностики лагов в играх?

Для анализа производительности в играх рекомендуется отслеживать:

1. GPU Engine → Utilization Percentage (загрузка видеокарты)

2. Память → Available MBytes (свободная оперативная память)

3. Процессор → % загрузки процессора (по ядрам)

4. Физический диск → % активного времени (нагрузка на HDD/SSD)

5. Сеть → Байт/сек (если игра онлайн)

Критичный нюанс: системный монитор показывает данные в реальном времени, но не сохраняет их автоматически. Если вам нужна история за несколько дней — настройте Журналы и оповещения производительности (об этом ниже).

Диагностика типичных проблем: что искать в системном мониторе

Системный монитор — это не просто наблюдательный инструмент, а мощное средство диагностики. Рассмотрим, какие параметры помогут выявить распространённые проблемы:

Проблема Какие счётчики добавить Что считать нормой Что делать, если значения критичные
Зависания системы Процессор → % загрузки, Память → Страниц/сек Загрузка CPU < 80%, страничные ошибки < 10 в сек Проверьте автозагрузку, обновления драйверов, вирусы
Медленная работа HDD Физический диск → Средняя длина очереди, % активного времени Очередь < 2, активность < 50% Дефрагментация, проверка на ошибки (chkdsk), замена на SSD
Перегрев ноутбука Температура процессора (если поддерживается), Процессор → % загрузки Температура < 80°C при нагрузке Очистка от пыли, замена термопасты, проверка кулера
Лаги в играх GPU Engine → Utilization, Память → Доступно МБ Загрузка GPU 90-100%, свободной памяти > 20% Снизьте настройки графики, закройте фоновые программы

Например, если вы заметили, что компьютер начинает "тормозить" при открытии нескольких вкладок в браузере, добавьте счётчики Память → Доступно МБ и Процесс → % загрузки процессора (выберите процесс chrome.exe или firefox.exe). Если свободной памяти остаётся менее 10%, это сигнал о необходимости увеличить объём ОЗУ или оптимизировать работу браузера (например, отключить ненужные расширения).

💡

Если системный монитор показывает 100% загрузку диска без видимых причин, проверьте службу Superfetch (или SysMain в Windows 10/11). Её отключение часто решает проблему.

Создание пользовательских отчётов: как сохранить данные для анализа

Одной из самых полезных функций системного монитора является возможность создания пользовательских отчётов. Это позволяет фиксировать данные о производительности за длительный период (например, несколько дней) и анализировать их позже. Вот как это сделать:

  1. Откройте системный монитор и перейдите в раздел Наборы сборщиков данных → Пользовательские.
  2. Нажмите правой кнопкой мыши на пустом месте и выберите Создать → Набор сборщиков данных.
  3. Задайте имя (например, Мониторинг игры) и выберите Создать вручную.
  4. Добавьте нужные счётчики (как описывалось выше) и укажите интервал сбора данных (например, каждые 5 секунд).
  5. На вкладке Параметры остановки настройте условие завершения (по времени или размеру файла).
  6. Сохраните набор и запустите его двойным кликом. Данные будут записываться в файл .blg.

Готовый отчёт можно открыть позже в системном мониторе через раздел Отчёты → Пользовательские. Это особенно полезно для сравнения производительности до и после изменений в системе (например, после обновления драйверов или добавления оперативной памяти).

Обратите внимание: длительный сбор данных с малым интервалом (например, каждую секунду) может создать большую нагрузку на диск. Если вам не нужна высокая детализация, установите интервал в 10-15 секунд.

Анализ отчётов: как читать графики и находить узкие места

Собранные данные бесполезны, если вы не умеете их интерпретировать. Рассмотрим, на что обращать внимание при анализе отчётов:

  • 📉 Пиковые нагрузки: резкие всплески на графике (например, загрузка CPU до 100%) могут указывать на фоновые процессы или вирусы. Сравните время пиков с запущенными программами.
  • 🔄 Циклические паттерны: повторяющиеся колебания (например, каждые 5 минут) часто связаны с запланированными задачами (Task Scheduler).
  • 🚨 Красные зоны: если линия графика постоянно находится в верхней части (например, загрузка диска 90%), это сигнал о перегрузке ресурса.
  • ⏱️ Задержки: в отчётах о сети или диске длительные "плоскости" могут означать зависания (например, из-за плохого подключения или фрагментированного HDD).

Для удобства используйте функцию Выделить (кнопка с лупой на панели инструментов). Она позволяет увеличить интересующий вас участок графика. Например, если игра начала лагать в определённый момент, найдите этот временной отрезок и проверьте, какой ресурс был перегружен: CPU, GPU или RAM.

💡

Если загрузка процессора достигает 100% при простых задачах (например, просмотре видео), скорее всего, проблема в драйверах или фоновых процессах. Проверьте диспетчер задач на наличие подозрительных приложений.

Для более глубокого анализа экспортируйте данные в CSV (через правый клик на графике → Сохранить данные как). Этот файл можно открыть в Excel или Google Sheets и построить собственные графики с дополнительными вычислениями (например, средние значения за час).

Продвинутые функции: оповещения и автоматический мониторинг

Системный монитор умеет не только собирать данные, но и автоматически оповещать вас о критических ситуациях. Например, можно настроить уведомление, если загрузка процессора превысит 90% дольше 5 минут или свободная память упадёт ниже 500 МБ. Вот как это работает:

  1. Перейдите в раздел Журналы и оповещения производительности → Оповещения.
  2. Нажмите правой кнопкой и выберите Новое оповещение о данных счётчика.
  3. Добавьте нужный счётчик (например, Память → Доступно МБ).
  4. Установите условие (например, меньше 500) и интервал проверки.
  5. Выберите действие: показать сообщение, запустить программу или отправить email (требует настройки SMTP).

Эта функция полезна для серверов или компьютеров, работающих в фоновом режиме (например, для майнинга или рендеринга). Вы можете настроить оповещение, которое запустит скрипт перезагрузки при перегреве или отправит вам письмо, если система перестанет отвечать.

Как настроить оповещение по email?

Для отправки email-оповещений необходимо:

1. Указать SMTP-сервер (например, smtp.gmail.com для Gmail).

2. Ввести логин/пароль от почтового ящика.

3. Настроить порт и шифрование (обычно порт 587 с TLS).

4. Убедиться, что антивирус не блокирует исходящие соединения.

Также системный монитор позволяет создавать логи производительности, которые автоматически запускаются при определённых условиях (например, при старте системы или по расписанию). Это удобно для долгосрочного мониторинга серверов или рабочих станций.

Частые ошибки и как их избежать

При работе с системным монитором пользователи часто сталкиваются с типичными проблемами. Вот самые распространённые из них и способы их решения:

  • 🔴 Нет данных на графике: убедитесь, что выбран правильный счётчик и он поддерживается вашей системой (например, температурные датчики могут отсутствовать на некоторых ноутбуках).
  • ⏳ Медленное обновление графика: уменьшите количество одновременно отображаемых счётчиков или увеличьте интервал обновления в свойствах.
  • 📁 Не сохраняются отчёты: проверьте права доступа к папке PerfLogs в корне диска C:. При необходимости запустите системный монитор от имени администратора.
  • 🔄 Счётчики сбрасываются после перезагрузки: это нормальное поведение. Чтобы сохранить настройки, экспортируйте их в файл .html через меню Действие → Сохранить настройки как.

Если системный монитор вовсе не запускается, проверьте целостность системных файлов. Откройте командную строку от имени администратора и выполните:

sfc /scannow

dism /online /cleanup-image /restorehealth

Эти команды восстановят повреждённые файлы Windows, которые могут влиять на работу инструмента.

💡

Если системный монитор не показывает данные по видеокарте, обновите драйверы GPU до последней версии. Часто это решает проблему с отсутствием счётчиков GPU Engine.

FAQ: ответы на популярные вопросы

Можно ли использовать системный монитор для проверки температуры процессора?

Да, но не на всех системах. Счётчик Температура процессора доступен только если ваш CPU и материнская плата поддерживают передачу этих данных через WMI. На многих ноутбуках и десктопах этот параметр может отсутствовать. В таком случае используйте сторонние утилиты вроде HWMonitor или Core Temp.

Как сравнить производительность до и после обновления Windows?

Создайте два пользовательских отчёта с одинаковыми счётчиками: один до обновления, другой — после. Затем откройте оба файла .blg в системном мониторе и сравните графики. Обратите внимание на средние значения загрузки CPU, использование памяти и активность диска. Значительные отклонения (более 20%) могут указывать на проблемы с новым обновлением.

Почему системный монитор показывает 100% загрузку диска, хотя я ничего не делаю?

Это типичная проблема Windows 10/11, связанная с фоновыми процессами, такими как индексирование поиска, обновления или служба Superfetch. Чтобы устранить её:

  1. Откройте диспетчер задач и найдите процесс, грузящий диск.
  2. Отключите индексирование: Службы → Windows Search → Остановить.
  3. Проверьте автозагрузку на наличие ненужных программ.
  4. Обновите драйверы контроллера диска через Диспетчер устройств.

Если проблема сохраняется, рассмотрите возможность замены HDD на SSD.

Можно ли мониторить удалённый компьютер через системный монитор?

Да, но для этого нужно настроить удалённый доступ. Откройте системный монитор, нажмите Действие → Подключиться к другому компьютеру и введите IP-адрес или имя удалённого ПК. Убедитесь, что на целевом компьютере:

  • Включена служба Удалённый реестр.
  • Открыт порт 135 TCP в брандмауэре.
  • У вас есть права администратора.

Этот метод работает только в локальной сети или через VPN.

Как экспортировать данные из системного монитора для анализа в Excel?

Чтобы экспортировать данные:

  1. Откройте нужный отчёт в системном мониторе.
  2. Нажмите правой кнопкой на график и выберите Сохранить данные как.
  3. Выберите формат Текстовый файл (CSV).
  4. Откройте полученный файл в Excel и используйте инструмент Текст по столбцам для корректного импорта.

В Excel вы сможете построить собственные графики, рассчитать средние значения и применить фильтры.
