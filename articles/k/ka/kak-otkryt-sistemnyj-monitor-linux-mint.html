Как открыть системный монитор в Linux Mint: все способы с пояснениями

Системный монитор в Linux Mint — это стандартная утилита для отслеживания производительности, аналогичная Диспетчеру задач в Windows. С её помощью можно контролировать загрузку процессора, оперативной памяти, сетевые соединения и активные процессы. Но не все пользователи знают, как быстро её запустить, особенно если привыкли к горячим клавишам из других ОС.

В этой статье мы разберём все возможные способы открытия системного монитора — от базовых (через графическое меню) до продвинутых (команды терминала и кастомизация). А ещё рассмотрим альтернативные инструменты, если стандартный монитор не устраивает по функционалу.

Особое внимание уделим ситуациям, когда утилита не запускается или отсутствует в системе. Это актуально для минимальных сборок Linux Mint или после некорректных обновлений. В конце статьи — FAQ с ответами на частые ошибки и советы по оптимизации мониторинга.

Способ 1: Запуск через главное меню

Самый очевидный метод — использовать стандартное меню приложений. Он подходит новичкам и тем, кто предпочитает графический интерфейс.

Откройте главное меню Linux Mint (значок в левом нижнем углу экрана или клавиша Super). В строке поиска введите «Системный монитор» или System Monitor. Приложение появится в результатах — кликните по нему.

Если поиск не срабатывает, попробуйте найти утилиту вручную:

  1. Откройте меню → Администрирование.
  2. Прокрутите список до пункта Системный монитор (в некоторых версиях может называться GNOME System Monitor).
⚠️ Внимание: В минимальных сборках Linux Mint Xfce системный монитор может отсутствовать. В этом случае его нужно доустановить через apt install gnome-system-monitor.

После первого запуска утилита добавится в список избранных приложений, и её можно будет быстро открывать из панели задач.

Способ 2: Горячие клавиши для быстрого доступа

Если вы часто используете системный монитор, имеет смысл назначить для него горячие клавиши. Это сэкономит время и избавит от необходимости искать приложение в меню.

Стандартной комбинации для системного монитора в Linux Mint нет, но её легко создать:

  1. Откройте меню приложений и найдите Системный монитор.
  2. Кликните правой кнопкой по значку → Добавить в избранное (если ещё не добавлено).
  3. Теперь кликните правой кнопкой по значку на панели задач → Добавить сочетание клавиш.
  4. Задайте удобную комбинацию, например Ctrl+Alt+M, и сохраните.

Если горячие клавиши не работают, проверьте конфликты с другими сочетаниями в настройках: Настройки → Клавиатура → Сочетания клавиш → Пользовательские сочетания.

💡

Чтобы не путаться в сочетаниях, используйте мнемонические комбинации. Например, Ctrl+Shift+Esc (как в Windows) можно переназначить на системный монитор через Настройки → Клавиатура → Сочетания клавиш → Системные → Показать системный монитор

Способ 3: Запуск через терминал

Терминал — универсальный инструмент для управления системой. Запустить системный монитор через него можно одной командой: 

gnome-system-monitor

Если команда не срабатывает, проверьте, установлен ли пакет: 

apt list --installed | grep gnome-system-monitor

Если пакет отсутствует, установите его:

sudo apt update && sudo apt install gnome-system-monitor

Для Linux Mint Xfce может потребоваться альтернативная утилита xfce4-taskmanager. Установите её командой: 

sudo apt install xfce4-taskmanager
⚠️ Внимание: В некоторых сборках Linux Mint на базе Cinnamon системный монитор может конфликтовать с встроенным апплетом мониторинга. Если утилита не открывается, попробуйте временно отключить апплеты через Настройки → Апплеты.

Через терминал также можно запускать системный монитор с правами суперпользователя (для просмотра всех процессов): 

sudo gnome-system-monitor

Но делайте это осторожно — случайное завершение системных процессов может привести к нестабильной работе ОС.

Способ 4: Добавление ярлыка на рабочий стол или панель

Если системный монитор используется часто, логично разместить его ярлык на рабочем столе или панели задач. Это избавит от необходимости искать приложение в меню.

Чтобы создать ярлык на рабочем столе:

  1. Кликните правой кнопкой по свободному месту на рабочем столе → Создать лаунчер.
  2. В поле Имя введите «Системный монитор».
  3. В поле Команда укажите gnome-system-monitor.
  4. Нажмите ОК и подтвердите создание.

Для добавления на панель задач:

  1. Запустите системный монитор любым из описанных способов.
  2. Кликните правой кнопкой по значку в панели → Закрепить на панели.

Если ярлык не работает, проверьте правильность команды. В некоторых случаях может потребоваться полный путь: 

/usr/bin/gnome-system-monitor

Способ 5: Альтернативные утилиты мониторинга

Стандартный системный монитор в Linux Mint предоставляет базовые функции, но для глубокого анализа системы могут понадобиться более мощные инструменты. Рассмотрим популярные альтернативы.

1. Htop — консольная утилита с расширенными возможностями: 

sudo apt install htop

htop

Преимущества: цветная визуализация, сортировка процессов, фильтрация.

2. Glances — универсальный монитор с веб-интерфейсом: 

sudo apt install glances

glances

Преимущества: поддержка удалённого мониторинга, экспорт данных в JSON/CSV.

3. Stacer — графический оптимизатор системы: 

sudo apt install stacer
Преимущества: очистка кэша, управление автозагрузкой, мониторинг температуры.
Утилита Тип интерфейса Особенности Установка
GNOME System Monitor Графический Базовый мониторинг, управление процессами По умолчанию
Htop Терминал Расширенная сортировка, цветные метрики sudo apt install htop
Glances Терминал/Веб Удалённый мониторинг, экспорт данных sudo apt install glances
Stacer Графический Оптимизация системы, очистка кэша sudo apt install stacer

Выбор утилиты зависит от задач:

  • 🔹 Для быстрого просмотра процессов — стандартный GNOME System Monitor.
  • 🔹 Для администрирования сервера — htop или glances.
  • 🔹 Для оптимизации системы — Stacer.
Как удалить ненужные утилиты мониторинга?

Чтобы освободить место, удалите неиспользуемые пакеты командой:

sudo apt remove --purge htop glances stacer

После удаления очистите зависимости:

sudo apt autoremove

Решение проблем с запуском системного монитора

Иногда системный монитор отказывается открываться или работает некорректно. Разберём типичные причины и способы их устранения.

1. Утилита отсутствует в системе.

В минимальных сборках Linux Mint (например, Xfce) системный монитор может не устанавливаться по умолчанию. Решение: 

sudo apt update && sudo apt install gnome-system-monitor

2. Конфликт с апплетами мониторинга.

В Cinnamon апплеты на панели (например, System Monitor Applet) могут блокировать запуск полноценной утилиты. Отключите их через: Настройки → Апплеты.

3. Повреждённые конфигурационные файлы.

Если системный монитор открывается, но не отображает данные, сбросьте его настройки: 

dconf reset -f /org/gnome/gnome-system-monitor/

4. Нехватка прав для просмотра процессов.

Если некоторые процессы не отображаются, запустите утилиту с правами sudo: 

sudo gnome-system-monitor
⚠️ Внимание: Завершение процессов от имени суперпользователя может привести к краху системы. Используйте эту команду только для диагностики, не изменяйте системные процессы.
💡

Если системный монитор не запускается после обновления Linux Mint, проверьте целостность пакетов командой sudo apt --fix-broken install

Настройка системного монитора под свои задачи

Стандартный интерфейс системного монитора можно адаптировать для удобства. Например, изменить отображаемые метрики или добавить новые вкладки.

1. Изменение отображаемых столбцов.

Вкладка Процессы позволяет настраивать видимые параметры:

  1. Кликните правой кнопкой по заголовку любого столбца.
  2. Выберите Показать столбцы.
  3. Отметьте нужные параметры (например, Nice, Приоритет, Канал ввода-вывода).

2. Мониторинг файловой системы.

Во вкладке Файловые системы можно отслеживать загрузку дисков. Чтобы добавить новую точку монтирования:

  1. Перейдите в Правка → Настройки.
  2. Во вкладке Файловые системы нажмите Добавить.
  3. Укажите путь к монтированному разделу (например, /mnt/data).

3. Настройка оповещений.

Системный монитор не поддерживает уведомления, но их можно реализовать через cron и скрипты. Например, чтобы получать предупреждение при загрузке CPU > 90%: 

#!/bin/bash

LOAD=$(uptime | awk '{print $10}' | cut -d',' -f1)


if [ $(echo "$LOAD > 9.0" | bc) -eq 1 ]; then


notify-send "Высокая нагрузка CPU" "Текущая загрузка: $LOAD"


fi

Сохраните скрипт в /usr/local/bin/check_cpu.sh, сделайте исполняемым (chmod +x) и добавьте в cron:

/5    * /usr/local/bin/check_cpu.sh

FAQ: Частые вопросы по системному монитору в Linux Mint

Почему системный монитор не показывает некоторые процессы?

Скорее всего, у вас недостаточно прав. Запустите утилиту с sudo: 

sudo gnome-system-monitor

Если это не помогло, проверьте, не скрыты ли процессы настройками фильтра (вкладка Процессы → Вид → Показать все процессы).

Можно ли изменить внешний вид системного монитора?

Да, но возможности ограничены. Вы можете:

  • 🔹 Изменить тему через Настройки → Внешний вид (затрагивает все приложения).
  • 🔹 Настроить отображаемые столбцы (правый клик по заголовку).

Для глубокой кастомизации потребуется редактировать CSS-темы GNOME, что не рекомендуется новичкам.

Как сохранить лог процессов для анализа?

Стандартный системный монитор не поддерживает экспорт логов, но вы можете использовать:

  1. htop с опцией логирования: 
    htop --log-file=/tmp/processes.log
  2. glances с экспортом в CSV: 
    glances --export csv --export-csv-file=/tmp/glances.csv
Системный монитор тормозит. Что делать?

Это может происходить из-за:

  • 🔹 Большого количества процессов — попробуйте отфильтровать список (вкладка Процессы → Вид → Мои процессы).
  • 🔹 Нехватки ресурсов — закройте ненужные приложения.
  • 🔹 Проблем с графическим драйвером — обновите драйверы через Драйверы устройств в меню.

Если проблема сохраняется, используйте лёгкие альтернативы вроде xfce4-taskmanager.

Как мониторить температуру CPU/GPU?

Стандартный системный монитор не отображает температуру. Установите Psensor: 

sudo apt install psensor

Или используйте lm-sensors для консольного вывода: 

sudo apt install lm-sensors
