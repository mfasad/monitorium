Как полностью отключить монитор ресурсов в Windows: пошаговые методы

Монитор ресурсов (Resource Monitor) — встроенный инструмент Windows, который отслеживает использование CPU, оперативной памяти, дисков и сети. Хотя он полезен для диагностики, многие пользователи сталкиваются с ситуацией, когда процесс resmon.exe грузит систему без видимой причины — особенно на слабых ПК или ноутбуках с ограниченными ресурсами. В некоторых случаях он может потреблять до 10-15% CPU в фоновом режиме, даже когда вы не запускали его вручную.

В этой статье мы разберём 5 проверенных способов отключить монитор ресурсов — от временного завершения процесса до полного удаления системных триггеров, которые его активируют. Важно понимать, что простое закрытие окна Resource Monitor через крестик не решает проблему: процесс продолжает работать в фоне. Мы также объясним, почему монитор ресурсов может самопроизвольно запускаться при подключении второго монитора или после обновлений Windows, и как это предотвратить.

Почему монитор ресурсов грузит систему?

Процесс resmon.exe не является вирусом — это легитимный компонент Windows, но его поведение часто вызывает вопросы. Основные причины высокой нагрузки:

  • 🔄 Автоматический запуск при диагностике: Windows активирует монитор ресурсов, если detectирует проблемы с производительностью (например, при долгой загрузке ПК или ошибках приложений).
  • 🖥️ Подключение дополнительных мониторов: При расширении экрана система может запускать Resource Monitor для анализа графической подсистемы.
  • 📊 Фоновый сбор данных: Даже без видимого окна процесс продолжает логировать активность системы для отчётов в Панель управления → Администрирование → Просмотр событий.
  • 🔄 Ошибки в планировщике задач: Некоторые задачи (например, Microsoft Compatibility Appraiser) могут непреднамеренно триггерить запуск resmon.exe.

На ноутбуках с интегрированной графикой (Intel UHD Graphics, AMD Radeon Vega) монитор ресурсов может запускаться чаще из-за конфликтов с драйверами дисплея. Если вы заметили, что процесс активируется при подключении внешнего монитора через HDMI/DisplayPort, проблема скорее всего кроется в настройках графического адаптера.

⚠️ Внимание: Отключение монитора ресурсов лишит вас возможности детальной диагностики системы через стандартные инструменты Windows. Если вы часто анализируете производительность, рассмотрите альтернативы вроде Process Explorer или HWiNFO.

Способ 1: Завершение процесса через диспетчер задач

Самый быстрый, но временный метод — принудительное завершение resmon.exe. Этот способ подходит, если процесс случайно запустился и грузит систему здесь и сейчас.

  1. Нажмите Ctrl + Shift + Esc, чтобы открыть Диспетчер задач.
  2. Перейдите на вкладку Подробности (в Windows 10) или Процессы (в Windows 11).
  3. Найдите процесс resmon.exe в списке. Если его нет, нажмите на колонку Имя для сортировки.
  4. Кликните по процессу правой кнопкой и выберите Снять задачу.

Если процесс появляется снова через несколько минут, значит он запускается какой-то системной службой. В этом случае переходите к следующим способам.

Способ 2: Отключение через планировщик задач

Многие задачи Windows по умолчанию настроены на запуск монитора ресурсов для сбора диагностических данных. Отключим их:

  1. Откройте Планировщик задач через поиск Windows (введите taskschd.msc).
  2. Перейдите по пути: 
    Библиотека планировщика задач → Microsoft → Windows → ResourceExhaustionDetector
  3. В правой панели найдите задачи с названиями:
    • Notify
    • ComputeResource
    • MemoryResource
  • Кликните по каждой правой кнопкой → Отключить.

    • Также проверьте раздел Microsoft → Windows → Customer Experience Improvement Program — здесь могут быть задачи, связанные с отправкой данных о производительности в Microsoft. Отключите их, если не участвуете в программе улучшения Windows.

    ⚠️ Внимание: Некоторые задачи в планировщике критичны для стабильности системы. Не отключайте процессы с названиями Critical, Update или Security без предварительного изучения их назначения.

    Способ 3: Отключение сбора данных о производительности

    Windows по умолчанию собирает данные о работе системы для отчётов и диагностики. Эта функция может автоматически запускать Resource Monitor. Отключим её:

    1. Откройте Параметры Windows (Win + I) → Конфиденциальность → Диагностика и отзывы.
    2. В разделе Данные диагностики выберите Отправить обязательные данные диагностики (вместо Необязательные данные).
    3. Отключите опцию Улучшить ввод текста и рукописного ввода... (если есть).
    4. Перейдите в Панель управления → Система и безопасность → Администрирование → Просмотр событий.
    5. В левом меню выберите Журналы приложений и служб → Microsoft → Windows → Resource-Exhaustion-Detector.
    6. Кликните правой кнопкой по журналу → Свойства → установите Отключить журнал.

    Эти действия уменьшат количество триггеров для запуска resmon.exe, но не гарантируют полного отключения. Для радикального решения переходите к следующему способу.

    💡

    Если после отключения диагностики Windows начала работать нестабильно, верните настройки обратно. Некоторые функции (например, автоматическое восстановление после сбоев) зависят от этих данных.

    Способ 4: Удаление триггеров через реестр (для опытных пользователей)

    Некоторые параметры монитора ресурсов хранятся в реестре Windows. Их изменение позволит отключить автозапуск процесса. Внимание: неверные действия в реестре могут привести к сбоям системы. Сделайте резервную копию перед внесением изменений.

    1. Нажмите Win + R, введите regedit и подтвердите.
    2. Перейдите по пути: 
      HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Perflib
    3. Найдите параметр EnableResourceMonitor. Если его нет, создайте его:
      • Кликните правой кнопкой в правой панели → Создать → Параметр DWORD (32 бита).
      • Назовите его EnableResourceMonitor.
      • Установите значение 0.
  • Перезагрузите компьютер.

    • После этих действий монитор ресурсов не должен запускаться автоматически. Если проблема сохраняется, проверьте наличие вирусов или конфликтов с сторонним ПО (например, антивирусами типа Avast или Kaspersky, которые могут эмулировать системные процессы).

    Параметр реестра Расположение Значение для отключения Примечание
    EnableResourceMonitor HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Perflib 0 Отключает автозапуск resmon.exe
    EnableUltraLight HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Reliability 0 Отключает сбор данных надёжности
    DisablePerformanceCounters HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\PerfProc\Performance 1 Отключает счётчики производительности
    Что делать, если после изменений в реестре Windows не загружается?

    Если система не стартует после редактирования реестра, загрузитесь в Безопасном режиме (удерживайте F8 при включении ПК) и восстановите реестр из резервной копии. Для этого откройте regedit, выберите Файл → Импорт и укажите путь к сохранённому файлу .reg.

    Способ 5: Альтернативное решение — ограничение процессов через групповую политику

    Если вы используете Windows Pro или Enterprise, можно ограничить запуск resmon.exe через редактор локальной групповой политики. Этот метод не удаляет процесс, но блокирует его выполнение.

    1. Нажмите Win + R, введите gpedit.msc и нажмите Enter.
    2. Перейдите по пути: 
      Конфигурация компьютера → Административные шаблоны → Система → Диагностика → Задачи диагностики
    3. Найдите политику Microsoft-Windows-ResourceExhaustionDetector.
    4. Дважды кликните по ней → выберите Отключено → Применить.

    Также рекомендуется отключить политики:

    • Microsoft-Windows-Kernel-EventTracing (отвечает за трассировку событий)
    • Microsoft-Windows-Diagnosis-Scripted (скрипты диагностики)

      • Эти изменения вступят в силу после перезагрузки. Метод особенно эффективен в корпоративных сетях, где требуется централизованное управление производительностью ПК.

      💡

      Редактор групповой политики доступен только в Pro/Enterprise-версиях Windows. Для домашней версии (Home) используйте способы 1–4.

      Что делать, если монитор ресурсов запускается при подключении второго монитора?

      Если resmon.exe активируется именно при расширении экрана, проблема кроется в драйверах графического адаптера или настройках Windows для множественных дисплеев. Вот как это исправить:

      1. Обновите драйверы видеокарты:
        • Для NVIDIA: скачайте последнюю версию с сайта nvidia.com.
        • Для AMD: используйте утилиту AMD Adrenalin.
        • Для Intel: обновите через Intel Driver & Support Assistant.
    • Отключите аппаратное ускорение для дисплеев: 
      Параметры Windows → Система → Дисплей → Графика → Изменить параметры по умолчанию → Отключить аппаратное ускорение для GPU
    • Сбросьте настройки экрана: 
      Панель управления → Оборудование и звук → Экран → Настройка разрешения → Дополнительно → Сбросить настройки

      • Если проблема сохраняется, попробуйте подключить второй монитор через другой порт (например, DisplayPort вместо HDMI) или используйте адаптер с активным усилением сигнала.

      ⚠️ Внимание: На ноутбуках с гибридной графикой (NVIDIA Optimus или AMD Switchable Graphics) конфликты с монитором ресурсов встречаются чаще. В этом случае поможет полное удаление драйверов через Display Driver Uninstaller (DDU) с последующей чистой установкой.

      FAQ: Частые вопросы об отключении монитора ресурсов

      Можно ли полностью удалить файл resmon.exe?

      Технически да, но это крайне не рекомендуется. Файл resmon.exe является системным компонентом, и его удаление может привести к ошибкам при диагностике оборудования или обновлениях Windows. Вместо этого используйте способы, описанные в статье, чтобы отключить его запуск.

      Почему монитор ресурсов запускается при игре?

      Во время игр Windows может активировать Resource Monitor для мониторинга использования GPU/CPU, особенно если игра работает нестабильно (низкий FPS, лаги). Чтобы это предотвратить:

      1. Отключите фоновый режим в настройках игры (если есть).
      2. Добавьте игру в исключения Центра безопасности Windows.
      3. Запускайте игру в режиме Высокая производительность (через Параметры графики в Windows).

    Как вернуть монитор ресурсов, если он понадобился?

    Если вы отключили процесс через реестр или групповую политику, но вдруг потребовалось его использовать:

    1. Для реестра: верните значение EnableResourceMonitor на 1.
    2. Для групповой политики: включите обратно политики в gpedit.msc.
    3. Запустите вручную: нажмите Win + R → введите resmon → Enter.

    Есть ли альтернативы монитору ресурсов?

    Да, если вам нужны более мощные инструменты для анализа системы, рассмотрите:

    • Process Explorer (от Microsoft) — расширенная версия диспетчера задач.
    • HWiNFO — детальная информация о железе и температурах.
    • Open Hardware Monitor — мониторинг напряжений и скорости вентиляторов.
    • GlassWire — анализ сетевой активности.

      • Эти программы не интегрированы в систему, поэтому не будут запускаться без вашего ведома.

    Может ли антивирус блокировать отключение монитора ресурсов?

    Да, некоторые антивирусы (например, Avast, ESET NOD32) могут воспринимать изменения в реестре или планировщике задач как подозрительную активность. Чтобы избежать блокировки:

    1. Временно отключите защиту антивируса.
    2. Добавьте regedit.exe и taskschd.msc в исключения.
    3. Используйте Безопасный режим для внесения изменений.
